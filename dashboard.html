DriftWatch

Autonomous MLOps platform for fraud detection with policy-driven drift response. Monitors feature & concept drift, triggers automated retraining, and promotes new model versions to production without manual intervention.

Quick start

1
Install Docker Desktop and make sure it is running. That's the only dependency — everything else runs in containers.
2
Start all services: make up — builds images on first run, starts MLflow, API, Prometheus, and Grafana, then opens this dashboard.
3
Run a full end-to-end demo (pick one):
make demo-drift-feature  — gradual shift in transaction distributions
make demo-drift-concept  — same features, different fraud patterns
make demo-black-friday   — sudden spike shock event
Each demo generates data → trains a model → promotes it → runs drift monitoring → triggers the control plane.
4
Check results in MLflow (model registry, training metrics) and call POST /predict via the Swagger UI to generate API traffic for Grafana/Prometheus.

Grafana and Prometheus only show data after POST /predict requests are made — the demo pipeline trains and monitors but does not call the inference API.  Local Python is only needed for development (make lint, make test).

Drift pipeline (run via make)

⚙️
Data Generation

Synthetic transaction datasets with configurable drift types: feature drift, concept drift, or shock events (Black Friday).

make gen-base make gen-feature make gen-blackfriday
🤖
Control Plane

Sentinel reads Evidently drift reports → Planner decides action → Release retrains and promotes the model automatically.

make control make monitor
🚀
End-to-End Demos

One-command demos that run the full pipeline: generate data → train → promote → monitor → control plane response.

make demo-drift-feature make demo-drift-concept make demo-black-friday
